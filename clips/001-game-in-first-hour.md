## Final Decision

採用案は Candidate A「作れる」。

核はこの一文。

```text

```

## Viewer Wound

- コードが読めないと、作れないと思っている。
- 言語、ライブラリ、環境、お作法が増えすぎて、追いつけないと思っている。
- AIに作らせたものは、自分の作品ではないと感じている。
- 最初の1時間で作品が動くとは信じていない。

## Target Persona Lock

この1話は、プログラミング学習を始めたい優等生だけに向けたものではない。

見る人は、スマホで30秒だけ見ている。
昔少し触ってやめた人、入門記事で止まった人、AIで何か作れるらしいが自分には関係ないと思っている人。

返り咲き層が離れた理由は、syntaxだけではない。
言語、フレームワーク、ライブラリ、環境構築、お作法が増え続け、「今から追いつくのは無理」と感じたことが大きい。

この人に「授業を受けよう」と言っても弱い。
先に、動いている画面を見せる。
そのあとで「これを最初の1時間で作る」と言う。

## Visual Direction

### Format

- 9:16 vertical
- 1080 x 1920
- 20〜35秒
- 外部素材なし
- 実在サービスのUIなし
- 実在ゲーム画面なし

### Look

- 背景は黒〜濃いグレー。
- 文字は白を基本にし、強調だけ明るい緑または水色。
- ゲーム画面は自作の抽象ドット風でよい。
- キャラは四角、三角、点で作る。既存ゲームに似せすぎない。
- 入力欄は特定サービスに似せない。角丸の白枠とカーソルだけでよい。

### Motion

1. 完成に近いゲーム画面から始める。
2. 自機が左右に動き、弾が1発だけ上へ動き、敵が1つ消える。
3. 「言語」「ライブラリ」「環境構築」のカードを積み、横にどける。
4. 入力欄に文字がタイプされる。
5. 作成中、起動中、修正中、プレビューの4状態を短く見せる。
6. ゲーム画面に戻り、視聴者が触っている感じを出す。
7. 最後に「動いた。」で締める。

## Small Action

視聴後にやることは1つだけ。

```text
sproutに、作りたい小さなゲームを普通の言葉で頼む。
```

プロンプト例。

```text
インベーダー風の小さなゲームを作って。
まず動くところまで進めて。
```

## Do Not Use

- 「作れるかも」
- 「初心者でも大丈夫」
- 「AI時代だから誰でも作れる」
- 仕様、要件定義、環境構築の説明、エラー、ログの話
- AIエージェントの機能説明
- Replit / Bolt / Lovable / GitHub Spark / MakeCode Arcade のUI、コピー、ロゴ、画面
- Space Invaders等の実画面、キャラ、音、ロゴ、名称をタイトルや誘引文句として使うこと
