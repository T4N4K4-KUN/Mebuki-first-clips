# 001 — 最初の1時間で、ゲームは動く

status: review

## Purpose

第一話の役割は、説明ではなくツカミ。
視聴者に「自分にも作れる」を最初に渡す。

## Core message

作れる。
最初の1時間で、ゲームは動く。

## Reference focus

001で参考にするのは、この5つだけ。

| Reference | 取り入れる核 | mebukiでの使い方 |
|---|---|---|
| Replit Agent | 普通の言葉でアプリは作れる | 「コードを書ける？」ではなく「何を作る？」から入る |
| Bolt.new | ビジョンに集中し、エラー説明で始めない | 冒頭を入力欄と作りたいものにする |
| Lovable | 自然言語からworking applicationへ進む | ゲーム、クイズ、シミュレーションを正当な入口にする |
| GitHub Spark | dream -> see -> ship | 作る、見える、公開できるの順番を画で見せる |
| MakeCode Arcade | 初手からゲーム | 動く画面の快感を最初に置く |

## Viewer wound

- コードが読めないと、作れないと思っている。
- 入門は文法や仕様から始めるものだと思っている。
- AIに作らせたものは、自分の作品ではないと感じている。
- 最初の1時間で作品が動くとは信じていない。

## What this clip must not do

- 「作れるかも」と弱くしない。
- 仕様、要件定義、エラー、ログの話をしない。
- AIエージェントの機能説明にしない。
- 外部サービスのUI、コピー、ゲーム画面を使わない。
- CS50/freeCodeCampを001の主参考にしない。後段に回す。

## Small action

sproutに、普通の言葉で小さなゲームを頼む。

```text
インベーダー風の小さなゲームを作って。
まず動くところまで進めて。
```

## Candidate A — 作れる

### Borrowed pattern

- Replit Agent: 普通の言葉から作る。
- Bolt.new: 「何を作る？」で始める。
- MakeCode Arcade: 最初からゲーム。

### 20-30 sec script

| Cut | Narration / Caption | Visual |
|---:|---|---|
| 1 | 作れる。 | 黒背景。中央に大きく「作れる。」 |
| 2 | 最初の1時間で、ゲームは動く。 | 入力欄。「何を作る？」 |
| 3 | たとえば、こう言うだけ。 | 自作チャットUIに「インベーダー風の小さなゲーム」 |
| 4 | コードを読めなくても、まず画面が出る。 | 抽象モックのゲーム画面。自機、敵、弾。 |
| 5 | 弾が動く。敵が動く。もう作品だ。 | 弾と敵が1回だけ動く。 |
| 6 | 最初の授業は、仕様じゃない。作るところから始めよう。 | 「今日の勝ち: ゲームが動いた」 |

### Why it works

一番強い言い切り。第一話のツカミとして、説明より先に「作れる」を置ける。

## Candidate B — 何を作る？

### Borrowed pattern

- Bolt.new: 作りたいものを聞く入口。
- Lovable: 自然言語からworking application。
- GitHub Spark: 作る、見える、届ける。

### 20-30 sec script

| Cut | Narration / Caption | Visual |
|---:|---|---|
| 1 | 最初の質問は、「何を勉強する？」じゃない。 | 「何を勉強する？」に× |
| 2 | 「何を作る？」 | 大きな入力欄。 |
| 3 | ゲームでいい。クイズでいい。小さなシミュレーションでもいい。 | 3つの小さな自作モックが並ぶ。 |
| 4 | 普通の言葉で言う。AIが作る。画面が動く。 | 入力 -> 作業中 -> プレビューの3段。 |
| 5 | その瞬間、あなたは見る側じゃなく、作る側になる。 | ゲーム画面が全画面になる。 |
| 6 | sproutの最初の1時間は、ゲームを動かすところから。 | 「何を作る？」が再表示。 |

### Why it works

初心者が逃げる単語を避けやすい。問いの置き換えだけで、第一話の思想が伝わる。

## Candidate C — 最初の授業でインベーダー

### Borrowed pattern

- MakeCode Arcade: 小ゲームで掴む。
- Replit Agent: Agentが作り、確認し、直す。
- GitHub Spark: 見えるものをすぐ動かす。

### 20-30 sec script

| Cut | Narration / Caption | Visual |
|---:|---|---|
| 1 | 最初の授業で、インベーダーを作る。 | レトロ風の自作ドット画面。 |
| 2 | 仕様書はまだ出さない。 | 「仕様書」を下にスライドして消す。 |
| 3 | まず言う。「小さなゲームを作って」 | 入力欄に短い依頼。 |
| 4 | AIが作る。動かす。直す。 | 抽象的な進行バーとチェック。 |
| 5 | 1時間後、画面の中で弾が飛ぶ。 | 弾が上へ動く。敵が消える。 |
| 6 | これが、今の開発の入口。 | 「作ってから学ぶ」 |

### Why it works

題材が最も具体的。sproutの「最初の授業でゲーム」を強く出せる。

## Recommendation

第一候補は Candidate A。
強い言い切りから入れるため、最初の一本のツカミとして一番ぶれない。

Candidate Bは、やわらかく始めたい場合。
Candidate Cは、sproutの授業内容を前面に出したい場合に使う。

## Rights note

外部サービスの画面、ロゴ、文言、実ゲーム画面は使わない。
画面はすべてmebukiの自作モックにする。
