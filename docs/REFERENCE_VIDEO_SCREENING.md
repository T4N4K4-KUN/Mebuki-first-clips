# REFERENCE_VIDEO_SCREENING

NotebookLM等に入れる前に、参考動画を選別するための基準。

今回、サービス/ツールの売り込み動画を多く入れてしまい、NotebookLM投入後の学びが薄かった。
今後は、動画を入れる前にこの表でふるいにかける。

## Minimum Criteria

候補動画は、まず対象ユーザーにとって有益・興味がわくかで見る。
そのうえで、次のうち最低3つを満たすこと。

| Criterion | Required evidence |
|---|---|
| 対象ユーザーにとって有益 | 初心者・返り咲き層が見た後に、試す理由や次の行動を得られる |
| 対象ユーザーが興味を持つ | 冒頭または成果物が「見たい」「触りたい」「自分もできそう」につながる |
| 冒頭3秒で止める力がある | 実際の冒頭秒数と画面/セリフ |
| 視聴者の心理変化がある | 「自分もできそう」「戻れそう」につながる場面 |
| 具体的な成果物が見える | ゲーム、アプリ、作品、投稿物など |
| 作る前後の差が見える | before -> after が動画内で確認できる |
| 初心者/返り咲き層の障壁を下げる | 環境、道具、情報量、失敗、再開への導線 |
| mebukiへ移植できる構造が2つ以上ある | 例: 結果先出し、ユーザー投稿紹介、1か所改造 |

## Reject Criteria

次は原則としてNotebookLMへ入れない。

- 対象ユーザーが見ても得がない。
- 対象ユーザーの興味より、作り手やサービス側の都合が前に出ている。
- サービス機能一覧の読み上げが中心。
- アフィリエイト/宣伝/比較煽りが中心。
- DB、API key、認証、課金、デプロイなど、001には重い話が中心。
- 30秒見ても、何が完成したか分からない。
- 台詞を抜いても、mebukiに移植できる構造が1つ以下。
- 公式/非公式、出典、秒数ログが曖昧。

## 001 Priority

001では、次の動画を優先する。

- 初心者・返り咲き層が「今なら戻れそう」「まず触れそう」と感じる。
- 見た後の行動が小さい。例: 1プロンプト入れる、1ゲームを触る、1か所だけ変える。
- ユーザー投稿作品、学生作品、初心者作品を紹介している。
- 最初に動くゲームや作品を見せている。
- 「既存作品から着想を得て作った」ことを肯定している。
- 作ったあとに中を見て、1か所だけ変える。
- ツールの売り込みより、作った人・触った人の変化が見える。
