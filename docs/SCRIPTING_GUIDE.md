# SCRIPTING_GUIDE

この文書は、mebuki starter clips v0 の脚本を作るときの手順と品質基準。
Codex / Claude Code は、白紙から完成台本を書いてはいけない。

## required process

1. `docs/RESEARCH_BASELINE.md` を読む。
2. `docs/CLIP_BRIEFS_V0.md` の対象clipを読む。
3. 視聴者の状態を1文で書く。
4. そのclipで下げる心理負荷を1つに絞る。
5. 視聴後の小さな行動を1つに絞る。
6. 20〜35秒の候補を2〜3案だけ出す。
7. 各候補を `docs/HUMAN_REVIEW_CHECKLIST.md` で自己評価する。
8. 人間が選ぶまで、最終版と断定しない。

## script shape

各候補は、3〜5カットにする。

- Cut 1: 視聴者の不安・詰まり
- Cut 2: 見方の小さな変更
- Cut 3: 今日の小さな行動
- Cut 4: 戻ってこられる感覚

または、次の4文構成にする。

- Hook: 今の不安を言い当てる
- Reframe: その不安の意味を変える
- Small action: 今日やる1つの行動
- Close: 小さな勝ちとして閉じる

## candidate format

各候補は次の形式にする。

```md
## Script candidate A — <狙い>

### Viewer moment
<この動画を見る人が、いつ・どんな気分で見るか>

### Cuts
1. <画面/字幕/ナレーション>
2. <画面/字幕/ナレーション>
3. <画面/字幕/ナレーション>
4. <画面/字幕/ナレーション>

### Small action
<視聴後にやる1つの行動>

### Why this may hit
<どの心理負荷を下げるか>

### Risk
<説教臭い、抽象的、刺さらない、などのリスク>

### Self review
- Specific human: /5
- Shame reduction: /5
- Small action: /5
- Rights safety: /5
- Brevity: /5
```

## language rules

使う言葉：短い、具体的、行動に落ちる、押し付けない、失敗者扱いしない、「今日」に着地する。

避ける言葉：

- 頑張ろう。
- 未来を変えよう。
- 誰でも簡単。
- あなたならできる。
- 挑戦し続けよう。
- 夢を叶えよう。
- プログラミングは簡単。
- 努力すれば必ず報われる。

これらは、それだけでは人間に刺さらない。使う場合も、必ず具体行動とセットにする。

## anti-patterns

### 名言化

「エラーは成長の証。あきらめなければ、あなたは必ず前に進める。」

不合格理由：誰のどの場面にも刺さらない。次の行動がない。

### 説教化

「毎日コツコツ学ぶことが大切です。まずは継続しましょう。」

不合格理由：できない人がすでに知っていることを言っているだけ。

### 内輪思想化

「mebukiは静かに優しく自然に広がる思想です。」

不合格理由：視聴者の課題ではない。repo内の作業思想を台本に混ぜない。

### AIっぽい励まし

「あなたの可能性は無限です。今日から新しい一歩を踏み出しましょう。」

不合格理由：薄い。誰にでも言える。

## final rule

完成台本を書こうとする前に、必ず候補を出す。候補が人間レビューを通るまで、`final` や `approved` と書かない。
