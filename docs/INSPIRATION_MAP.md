# INSPIRATION MAP — mebuki clips v0

このrepoでいう「参考にする」は、映像・音声・字幕・台本・図解・キャラ・サムネをコピーすることではない。
取り入れる対象は、**アイデア、心理トリガー、構成、テンポ、学習導線、視聴者への立ち位置**である。

目的は、Codex/Claude Codeが白紙から「それっぽい励まし」を創作しないようにすること。
6本の短い動画は、以下の型から着想を得てmebuki向けに作り直す。

## 原則

- 表現は自作する。
- ただし、優れた型は遠慮なく取り入れる。
- 「何となく良いことを言う」は禁止。
- 「誰の、どの不安に、どんな視点反転を起こすか」を先に決める。
- 1本につき、参考にする型は2〜3個まで。混ぜすぎない。
- 6本以外に増やさない。

## 参考の強度

| Level | 内容 | このrepoでの扱い |
|---:|---|---|
| 0 | 雰囲気だけ見た | 弱い。脚本に入れるな |
| 1 | テーマを参考にした | まだ弱い |
| 2 | 構成を取り入れた | 採用可 |
| 3 | 心理トリガー・視点反転を取り入れた | 本命 |
| 4 | 台詞・図・映像・編集を真似た | 禁止 |

狙うのは **Level 2〜3**。
Level 4は禁止。YouTube的にもmebuki的にも弱い。

# 取り入れる対象一覧

## 1. AI app builderから参考にする

### 取り入れるアイデア

- 入口は「仕様を決める」ではなく「何を作る？」にする。
- 普通の言葉で言うと、Agentが作る・確認する・直す・動かす。
- コードやエラーを先に見せず、動く画面を先に見せる。
- Dream it -> see it -> ship it の順番にする。
- 作れるかも、ではなく「作れる」と言い切る。

### mebuki変換

- 最初の授業で、ゲームを作る。
- インベーダー系、追いかける系、クイズ、シミュレーションのように、画面が動く題材から入る。
- 最初の1時間の勝利条件は「説明できる」ではなく「動いた」。
- 001では、仕様、要件、エラー解説、環境構築の細部に入らない。

### 使うclip

- 001 最初の1時間で、ゲームは動く
- 004 AIに聞いていい。検索していい。
- 005 理解より、まず動くものを作る
- 006 ログを残せる人は、戻ってこられる

### コピーしないもの

- Replit / Bolt / Lovable / GitHub Spark / MakeCode Arcade のUI、コピー、ロゴ
- 公式ドキュメントやLPの文面
- デモ画面
- 事例アプリの見た目
- プロンプト例の丸写し

## 2. ゲーム導入から参考にする

### 取り入れるアイデア

- 最初からゲームにする。
- 画面が動く快感を、理解より先に置く。
- 小さなルール、すぐ見える結果、短い反復で進める。
- 失敗してもすぐやり直せる題材にする。

### mebuki変換

- 001の題材は、Space Invaders系 / Galga系 / Pizza chase系のどれかに寄せる。
- 外部ゲーム画面は使わない。mebuki用の抽象モックにする。
- 「ゲームを作れる」は初心者向けの弱い目標ではなく、AIエージェント時代のツカミ。

### 使うclip

- 001 最初の1時間で、ゲームは動く

### コピーしないもの

- MakeCode Arcadeのゲーム名、キャラ、画面
- チュートリアル構成
- ブロック配置やコード例

## 3. freeCodeCamp / Odin / Exercismから参考にする

### 取り入れるアイデア

- 学習を「知識の順番」ではなく「作れるものの順番」にする。
- 長い教材を、小さな到達点へ分解する。
- 無料・自走・プロジェクト・認定の階段を作る。
- 「読む」より「手を動かす」を先に置く。
- 1ステップごとに小さな達成感を作る。

### mebuki変換

- 1本の動画に1行動だけ入れる。
- 「理解した？」ではなく「何か1つ動いた？」で判定する。
- index.htmlを作る、ボタンを置く、console.logする、READMEに1行書く、などの小さい勝利にする。
- 長いロードマップではなく「今日の1手」を提示する。

### 使うclip

- 002 今日は5分だけ触れば勝ち
- 005 理解より、まず動くものを作る
- 006 ログを残せる人は、戻ってこられる

### コピーしないもの

- 記事本文
- YouTube動画の字幕
- 課題文
- カリキュラム文面
- 説明の順番の丸写し

## 4. TED-Edから参考にする

### 取り入れるアイデア

- 冒頭を「説明」ではなく「問い」で始める。
- 30秒でも小さな物語にする。
- 最後に視点をひっくり返す。
- 「情報を増やす」のではなく「見方を変える」。
- 1本1テーマ。詰め込まない。

### mebuki変換

- 「作る側に回るのは、いつからか？」
- 「今日やる気がないのは、本当に意志が弱いからか？」
- 「昔やめた経験は、負債なのか？」
- 「AIに聞くのはズルなのか？」
- 「理解してから作るべきなのか？」

### 使うclip

- 003 昔やめた人は、弱い人じゃない
- 004 AIに聞いていい。検索していい。
- 005 理解より、まず動くものを作る

### コピーしないもの

- アニメーション
- ナレーション文
- 動画タイトル
- 図解
- 問いの言い回し

## 5. Crash Courseから参考にする

### 取り入れるアイデア

- 冒頭3秒で止める。
- 途中で視覚的な切り替えを入れる。
- 真面目な内容でもテンポを落としすぎない。
- 章立ての気配を出す。
- 退屈しそうな瞬間に、短いツッコミや視点変更を入れる。

### mebuki変換

- 1カット3〜5秒。
- 1画面1文。
- 1動画につき画面切替5〜7回。
- 画面上の言葉は短く、ナレーションが少し補う。
- ただし初期v0では凝った編集をしない。静止画＋テキスト移動で十分。

### 使うclip

- 全6本の編集テンポ

### コピーしないもの

- ジョークの型そのもの
- キャラ演出
- BGMや効果音
- 速すぎる情報密度

- 「ボタンが1つ表示されればOK」
- 「エラー文の1行目だけ見ればOK」
- 「READMEに次の一手が残ればOK」

### 使うclip

- 005 理解より、まず動くものを作る
- 006 ログを残せる人は、戻ってこられる

## 9. 瞑想・メンタル系ショートから参考にする

### 取り入れるアイデア

- できなかった自分を責めるより、戻る動作を設計する。
- 雑念が出たら失敗ではなく、気づいて戻るのが練習。
- 比較を止め、現在地に戻す。
- 小さな呼吸、短い停止、やり直しを肯定する。

### mebuki変換

- 昔やめた経験は、弱さではなく、詰まりやすい場所の記録。
- 1週間空いたら終わりではなく、ログに戻れば再開。
- エラーで止まったら、エラー文に戻る。
- 迷ったらTODOに戻る。
- 戻れる人は強い。

### 使うclip

- 003 昔やめた人は、弱い人じゃない
- 006 ログを残せる人は、戻ってこられる

### コピーしないもの

- 癒しすぎる口調
- スピリチュアル寄りの表現
- 抽象的な自己肯定だけの台本

# Codexへの禁止

Codex/Claude Codeは、以下をしてはならない。

- 「頑張ればできる」だけの台本を書く。
- 「未来を変えよう」「一歩踏み出そう」だけの台本を書く。
- 視聴者の具体的な場面がない台本を書く。
- 6本を勝手に増やす。
- 既存海外動画の字幕・台本を日本語化する。
- 外部素材の図や映像を再現する。
- mebuki内部の戦略や思想を視聴者向け台本へ混ぜる。
- 完成稿を1案だけ出す。

Codex/Claude Codeは、必ず候補を複数出し、人間が選べる状態にする。
