# mebuki clips v0 decision board

このファイルは、最初の6本だけに集中するための意思決定ボード。
Codex/Claude Codeは、脚本を書く前に必ずこのファイルと `docs/INSPIRATION_MAP.md` を読む。

## 共通仕様

| 項目 | 指定 |
|---|---|
| format | 9:16 vertical short |
| size | 1080 x 1920 |
| length | 20〜35秒 |
| delivery | YouTube Shorts / Instagram Reels想定 |
| production | 静止画 + テキスト + 軽いモーションでよい |
| subtitles | 必須。音なしでも意味が通ること |
| voice | 任意。v0はテキストだけでもよい |
| external footage | 使わない |
| external scripts | 使わない |
| external diagrams | 使わない |
| first goal | 6本全部の完成ではなく、まず1本の刺さりを検証 |

# 001 — エラーが出たら、始まっている

## 参考にする型

- CS50: 難しさを参加儀式に変える
- TED-Ed: 冒頭の問い → 視点反転
- Crash Course: 赤い画面などの強いビジュアルで3秒停止

## 視聴者の傷

- エラーが出ると「自分には向いていない」と感じる
- 赤い文字を見た瞬間に閉じたくなる
- AIに聞く前に、自分を責める

## 3秒hook候補

- 赤い文字が出た。失敗？違う。
- エラーが出た日、ようやく開発が始まった。
- その赤い文字、拒絶じゃない。

## 視点反転

エラーは「お前はダメ」ではない。
プログラムが「ここを見て」と言っているだけ。

## 最後の小さな行動

エラー文の1行目だけコピーして、AIにこう聞く。

```text
このエラーを初心者向けに、原因と次の1手に分けて説明して。
```

## 画面案

1. 赤いエラー風の画面
2. 大きく「失敗？」
3. 反転して「会話開始」
4. エラー文の1行目にハイライト
5. AIへの質問文
6. 「今日はここまでで勝ち」

## 禁止

- 「エラーを恐れるな」だけで終わる
- 「プロもエラーを出す」だけで終わる
- エラー画面を本物の外部教材から持ってくる
- コードの正確な解説に深入りする

## 良い台本の匂い

- 視聴者が「昨日の自分だ」と感じる
- エラーを人格否定から切り離せる
- 次の行動が1つだけある

# 002 — 今日は5分だけ触れば勝ち

## 参考にする型

- 筋トレ: 1回でOK
- 英語学習: 1日1フレーズ
- freeCodeCamp: 小さい自走ステップ

## 視聴者の傷

- 勉強時間を大きく考えて着手できない
- 「今日はちゃんとやれないからやめる」となる
- 開発環境を開くまでが重い

## 3秒hook候補

- 今日、勉強しなくていい。
- 5分だけ開けたら勝ち。
- やる気がない日は、量を捨てる。

## 視点反転

継続は「長くやること」ではない。
戻る場所に今日も触ること。

## 最後の小さな行動

repoを開いて、READMEかTODOに1行だけ書く。

```text
今日の次の1手: ______
```

## 画面案

1. 机の前で止まるイメージ
2. 「1時間やる？」を×
3. 「5分だけ開く」を○
4. TODOに1行
5. 閉じてもOK
6. 「触れたら勝ち」

## 禁止

- 根性論
- 毎日継続しようという圧
- 朝活・習慣術に寄せすぎる
- 成功者風の煽り

## 良い台本の匂い

- 視聴後に本当に5分だけ触りたくなる
- 罪悪感を減らす
- 行動が小さすぎるくらい小さい

# 003 — 昔やめた人は、弱い人じゃない

## 参考にする型

- 瞑想: 逸れたら戻るのが練習
- Khan Academy: 恥を取り除く
- TED-Ed: 過去の意味を変える

## 視聴者の傷

- 過去にプログラミングを諦めた経験がある
- 「自分は続かない人間だ」と思っている
- 再開するのが恥ずかしい

## 3秒hook候補

- 昔やめた人へ。
- やめた経験は、弱さじゃない。
- 一度離れた人ほど、戻り方を知れる。

## 視点反転

挫折は「向いていない証拠」ではない。
どこで詰まったかを教える地図。

## 最後の小さな行動

昔止まった理由を1行だけ書く。

```text
前に止まった場所: ______
```

## 画面案

1. 古いフォルダ / 放置されたrepo
2. 「また無理かも」
3. 「違う。これは地図」
4. 止まった理由を1行
5. 次はそこを小さくする
6. 「戻る人は弱くない」

## 禁止

- 「あなたならできる」だけ
- 過去の挫折を美談にしすぎる
- 返り咲き層を中年いじりする
- 情緒だけで行動がない

## 良い台本の匂い

- 再開への恥が少し減る
- 「もう一回開くか」と思える
- 過去の失敗を材料化できる

# 004 — AIに聞いていい。検索していい。

## 参考にする型

- CS50: Q&A/ツールに頼ることを普通にする
- freeCodeCamp: 自走は孤独ではない
- TED-Ed: 「ズル？」という問いから反転

## 視聴者の傷

- AIに聞くとズルだと思っている
- 検索する自分を低く見ている
- 自力で全部理解できないと向いていないと思う

## 3秒hook候補

- AIに聞いたらズル？
- 検索している時点で負け？
- ひとりで解ける人だけが本物、ではない。

## 視点反転

現代の開発は、暗記勝負ではない。
問いを立て、答えを検証し、動かす仕事。

## 最後の小さな行動

AIに丸投げではなく、分解して聞く。

```text
この作業を初心者向けに、次の1手だけに分解して。
```

## 画面案

1. 「AIに聞く = ズル？」大文字
2. ×「丸投げ」
3. ○「分解して聞く」
4. プロンプト例
5. 実行する小さな次の1手
6. 「聞ける人は進める」

## 禁止

- AI万能論
- AIに全部作らせればいい
- 人間が学ばなくていい
- ツール名の宣伝っぽさ

## 良い台本の匂い

- AI利用への罪悪感が減る
- でも丸投げではなく、自分の次の行動が残る
- 返り咲き層にも現代環境として自然に伝わる

# 005 — 理解より、まず動くものを作る

## 参考にする型

- freeCodeCamp: project-first
- Khan Academy: ここまででOK
- 英語学習: 文法前に1フレーズ使う

## 視聴者の傷

- 理解してからでないと作ってはいけないと思う
- 入門記事を読み続けて手が動かない
- 完璧な理解を目指して止まる

## 3秒hook候補

- 理解してから作る？それで止まることがある。
- まず動くもの。理解は後から追いつく。
- 入門記事を閉じて、1つ動かそう。

## 視点反転

理解は入口ではなく、手を動かした後に深まることが多い。
最初のゴールは「説明できる」ではなく「触ったことがある」。

## 最後の小さな行動

サンプルを1つ動かし、1文字だけ変える。

```text
Hello を Mebuki に変える。
```

## 画面案

1. 入門記事が何枚も開いている
2. 「まだ理解できない」
3. 画面を閉じる
4. 小さいサンプルが動く
5. 1文字だけ変更
6. 「今日は理解より接触」

## 禁止

- 基礎を軽視する言い方
- コピペだけでいいという誤解
- 大きなアプリを作ろうとする
- 解説動画化しすぎる

## 良い台本の匂い

- 完璧主義が少し緩む
- 実行・変更・確認の小ループが伝わる
- その場で試せる

# 006 — ログを残せる人は、戻ってこられる

## 参考にする型

- mebuki独自: RUN/TODOで戻る導線を作る
- 瞑想: 逸れても戻る
- 筋トレ: 継続は再開能力

## 視聴者の傷

- 一度間が空くと何をしていたか忘れる
- 再開コストが高くてまたやめる
- 自分は継続力がないと思う

## 3秒hook候補

- 才能より強いものがある。
- 1週間空いても戻れる人がいる。
- 昨日の自分へのメモが、明日の自分を助ける。

## 視点反転

継続力とは、毎日できる才能ではない。
空いたあとに戻れる導線を残すこと。

## 最後の小さな行動

作業を終える前に、次の1手だけ書く。

```text
次にやること: ______
```

## 画面案

1. 数日後にrepoを開く
2. 「何してたっけ？」
3. TODOに「次にやること」
4. すぐ再開
5. 「ログは未来の自分への橋」
6. 「戻れたら勝ち」

## 禁止

- mebukiの内輪思想を語る
- RUN.mdの仕様説明に寄りすぎる
- 開発管理の話だけにする
- 継続できない人を責める

## 良い台本の匂い

- 明日忘れる自分を前提にできる
- ログを書く意味が感情で分かる
- mebukiの独自性が自然に出る
