# Vids prep — Codex app + Mebuki で最初の1時間ゲーム制作

checked_date: 2026-06-15

## Purpose

このメモは、Google Vidsで作る講義ビデオの準備資料。

目的は、初心者・返り咲き層に、

```text
余計なサービスやアプリを増やさず、
Codex app と Mebuki だけで、
最初の1時間に小さなゲームを完成させる
```

という1st STEPを見せること。

これは6本以外の大量企画ではない。001 `最初の1時間で、ゲームは動く` を、講義ビデオ用に具体化する補助資料として扱う。

## Core Message

中心メッセージは1つ。

```text
最初の1時間は、勉強を増やす時間ではなく、動くものを1つ作る時間。
```

言い換え:

```text
最初から全部理解しない。
最初からサービスを増やさない。
知っているゲーム名から始めてよい。
Codex app と Mebukiで、小さい練習版まで削って、1つだけ動かす。
```

## Naming Policy

この講義では、最初の入口を必ず `ゲーム具体名風` にする。

理由:

- 初心者は、ゲーム仕様をゼロから語れない。
- いきなり `当たり判定`、`状態管理`、`スコア条件` のように言わせない。
- まず `スイカゲームみたいなもの`、`キャンディクラッシュみたいなもの`、`FarmVilleみたいな小さな畑` のように、知っている型から始める。
- そのあと、AIと一緒に `最初の1時間で作れる一番小さい版` へ削る。

ただし、作るものは練習用のミニ版。
外部ゲームの画面、名前、キャラ、音、ロゴを使って公開するものではない。

## Audience

主対象:

- プログラミングを始めたいが、環境構築やツール選びで止まる人。
- 昔少し触って、今の開発環境の多さに戻れない人。
- AIで作れるらしいが、自分が何を頼めばよいか分からない人。
- ゲームが好きな人だけでなく、カジュアルゲーム、育成、日常ツールに近い入口の方が入りやすい人。

視聴者の傷:

- 何をインストールすればいいか分からない。
- IDE、GitHub、Node、npm、ブラウザ、ホスティングなどが一気に出てきて怖い。
- AIに頼むと、何が自分の作品なのか分からなくなる。
- 有名ゲームの名前を出すと、権利的に危ないことをしている気がする。

視点反転:

```text
最初の1時間で必要なのは、開発者の全部入り環境ではない。
AIに伝わる小さい注文票と、動いたかを見られる画面。
```

## Product / Tool Premise

講義ビデオ内で使う前提:

- 使うものは `Codex app` と `Mebuki` に絞る。
- Codex appは、公式Windows docs上ではMicrosoft Storeからの入手、または `winget install Codex -s msstore` が案内されている。
- Codex appはWindowsネイティブではPowerShellで動き、Windows sandboxまたはWSL2構成を選べる。
- Git、Node.js、Python、GitHub CLIなどは「あるとよい開発者ツール」として公式docsに挙がっているが、1st STEPの講義では全部を先に説明しない。
- Mebuki側で、空のプロジェクト、作業ログ、次の一手、完成判定を見せる。

このビデオでは、他のAIアプリ、ノーコードサービス、素材サイト、動画生成ツール、ホスティングサービスを入口にしない。

## Reference Types To Use

参考にする型:

- Scratch公式系: 完成像を先に見せ、5つ程度の小さい動作に分解する。
- AI app builder系: `何を作る？` から始め、コード説明より動く画面を先に置く。
- PRD / user story / acceptance criteria系: 表では難しい言葉を出さず、裏側で `注文票` と `できた判定` に変換する。
- AI agent clarification系: いきなり作らせず、作る前に `足りない情報を3つだけ質問して` と頼む。
- 具体名風hook系: 初心者が知っているゲーム名やアプリ名を出し、AIと一緒に練習用ミニ版へ削る。

コピーしないもの:

- 外部動画の映像、字幕、台本、図解。
- Codex app公式画面のスクリーンショットや製品コピー。
- 有名ゲームの画面、キャラ、音、ロゴ、名称を作品名として使うこと。

## Game Candidate Set

講義ビデオでは、冒頭で3〜4案を見せる。
実演は1つに絞る。

| Viewer phrasing | Internal type | Why include | 1-hour fit | Risk | Handling |
|---|---|---|---:|---|---|
| Snaky Catみたいに、好きなキャラがアイテムを集めて長くなるゲーム | Snake / Snaky | 操作、得点、ゲームオーバーが分かりやすい。キャラ差し替えで自分ごと化しやすい。 | high | データ根拠はFarm/Match3より弱い | 実演第一候補 |
| FarmVilleや牧場ゲームみたいに、水をあげて収穫する小さな畑 | Farm / harvest | Family/Farm Simに近く、非ギーク層にもやさしい。クリックで変化が見える。 | high | 映像の派手さは弱い | 実演第二候補 |
| スイカゲームみたいに、同じものをくっつける落ちもの | Merge puzzle | ルールが直感的でカジュアル。見た目が柔らかい。 | medium | 物理挙動で沼る | 完全再現ではなく3種類だけ |
| キャンディクラッシュやパズドラみたいに、3つそろえるパズル | Match 3 | Match 3文脈で女性比率が高いジャンルに近い。 | medium-low | 消去、補充、連鎖で重い | 5x5、4種、連鎖なし |

今回の推奨:

```text
冒頭で4案を、必ず具体名風で見せる。
実演は「Snaky Catみたいに、好きなキャラがアイテムを集めて長くなるゲーム」。
代替実演は「FarmVilleや牧場ゲームみたいな、小さな畑」。
```

理由:

- 1時間完走の確度を優先する。
- 初心者に `ゲーム仕様を説明して` と言わない。
- Snaky Catみたいな、と言えば、操作と成長のイメージが先に立つ。
- FarmVilleや牧場ゲームみたいな、と言えば、戦闘なしでもゲーム/アプリになることが伝わる。
- スイカ風とMatch3はフックは強いが、初回講義の実装失敗リスクが高い。

## 60-Minute Lesson Run

### 0:00-3:00 — Hook

見せる画:

- 完成した小さいゲームが動く。
- 4案が具体名風カードで出る。
- `今日は、Codex app と Mebukiだけで進めます。`

言うこと:

```text
今日は、最初の1時間でゲームを1つ動かします。
文法の授業ではありません。
仕様を書け、という授業でもありません。
ツール探しの授業でもありません。
Codex app と Mebuki だけで進めます。

最初は、知っているゲーム名からで大丈夫です。
```

### 3:00-10:00 — Install Codex app

見せる画:

- 公式ページまたはMicrosoft Storeへ移動。
- Windowsなら `winget install Codex -s msstore` という選択肢があることを短く示す。
- インストール後、起動してサインインする。

言うこと:

```text
ここで入れるのはCodex appだけです。
GitやNodeは必要になったらCodexに確認させます。
最初から開発者セットを全部そろえようとしません。
```

注意:

- 実際の画面録画で公式UIを使う場合は、権利・利用規約・表示範囲を人間が確認する。
- Vids用の説明スライドでは、自作の抽象UIにする。

### 10:00-15:00 — Open Mebuki

見せる画:

- Mebukiの最初の画面。
- `今日のゴール`、`注文票`、`できた判定`、`ログ` の4ブロック。

言うこと:

```text
Mebukiで今日のゴールを1つにします。
完成は、公開ではありません。
最初の完成は、ローカルで動くことです。
```

### 15:00-22:00 — Choose One Small Game

見せる画:

- 4候補を具体名風で並べる。
- `Snaky Catみたいに、好きなキャラがアイテムを集めて長くなるゲーム` を選ぶ。
- 有名ゲームの名前は `型の説明` として使い、作品名にはしない。

最初に視聴者が言える言葉:

```text
Snaky Catみたいに、
好きなキャラがアイテムを集めて長くなるゲームを作りたい。
```

MebukiでAIと一緒に削った注文票:

```text
Snaky Catみたいに、好きなキャラがアイテムを集めて長くなるゲームを作りたい。
ただし、練習用のミニ版にしてください。
最初は丸いアイコンでよい。
矢印キーで動かす。
アイテムを取るとスコアが増えて体が長くなる。
壁か自分の体にぶつかったらゲームオーバー。
保存、ランキング、スマホ対応、効果音はまだ作らない。
作る前に、足りない情報があれば3つだけ質問してください。
```

### 22:00-30:00 — Ask Codex To Clarify

見せる画:

- Codexが質問する。
- 人間が短く答える。
- Mebukiに `決めたこと` が残る。

想定質問:

```text
1. 画面サイズは固定でよいですか？
2. ゲームはブラウザで動けばよいですか？
3. 見た目はシンプルな図形でよいですか？
```

答え:

```text
固定でよいです。
ブラウザで動けばよいです。
最初は図形でよいです。
```

### 30:00-45:00 — Let Codex Build

見せる画:

- Codexがファイルを作る。
- 必要なコマンドを実行する。
- エラーが出たらCodexに直させる。

言うこと:

```text
ここで大事なのは、エラーを自分で全部読めることではありません。
Codexに、今起きていることを説明させます。
そして、次の1手だけを実行します。
```

Codexへの指示:

```text
まずローカルで動く最小版を作ってください。
作った後に、起動方法と確認ポイントを3つだけ教えてください。
途中で判断が必要なら、勝手に決めずに質問してください。
```

### 45:00-55:00 — Verify The Game

できた判定:

```text
- 矢印キーで動く。
- アイテムを取るとスコアが増える。
- 壁か自分の体に当たるとゲームオーバーになる。
```

見せる画:

- 実際に動かす。
- 1つだけ見た目を変える。
- `丸` を `好きな色の丸` にする程度で止める。

言うこと:

```text
今日は、デザインを完成させません。
1か所だけ変えて、自分の作品に触った感覚を作ります。
```

### 55:00-60:00 — Leave A Return Path

Mebukiに残すログ:

```text
今日できたこと:
Snaky Catみたいな型を借りた、練習用ミニゲームがローカルで動いた。

次にやること:
キャラとアイテムの見た目を自分用に変える。

公開前の注意:
有名ゲームの名前、見た目、キャラ、音は使わない。
練習用から自分の作品へ変えてから見せる。
```

締め:

```text
最初の1時間の勝ちは、理解しきることではありません。
動いたものと、戻れるログが残ることです。
```

## Vids制作構成

Vids本体に入れる英語プロンプトは `prompts/vids/VIDS_SCENE_PROMPTS_EN.md`。

各スライド内に出す文字は、日本人の初回視聴を前提に日本語へ寄せる。

## Human Review Points

人間が選ぶ点:

1. 実演題材を `Snaky Catみたいな、好きなキャラがアイテムを集めて長くなるゲーム` にするか、`FarmVilleや牧場ゲームみたいな小さな畑` にするか。
2. Codex appインストールを実画面で見せるか、抽象モックで見せるか。
3. Mebukiの画面をどこまで実画面で見せるか。
4. 完成ゲームを公開しない前提にするか、ローカル確認だけにするか。

## Unresolved Rights / Policy Items

- Google Vids内のテンプレ、BGM、画像を使う場合の利用条件は未確認。
