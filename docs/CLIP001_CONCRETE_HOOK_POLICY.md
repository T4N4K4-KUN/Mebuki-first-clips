# CLIP001_CONCRETE_HOOK_POLICY

001のツカミ方針。

目的は「権利リスクの授業」ではない。
目的は、初心者・返り咲き層・非ギーク層に、

```text
自分の言葉で頼めば、最初の1時間で小さいものが動く
```

と感じてもらうこと。

## Core Policy

抽象名で逃げない。

初心者は、ゲームやアプリの仕様をゼロから説明できない。
だから、最初は具体名風で入る。

悪い言い方:

```text
隣接する同種アイコンを交換し、3連結以上を検出して消去するパズル
```

良い言い方:

```text
キャンディクラッシュみたいな、3つそろえるパズル
パズドラみたいな、かわいいアイコンをそろえるゲーム
スイカゲームみたいな、同じものをくっつけるゲーム
インベーダーみたいな、上から来る敵を撃つゲーム
```

ただし、作るのは練習用のミニ版。
売るものでも、公式を名乗るものでもない。

## Candidate Hooks

001冒頭で見せる候補。

| Candidate | Concrete phrasing | Why it works | Risk | 001 handling |
|---|---|---|---|---|
| インベーダー風 | インベーダーみたいに、上から来る敵を撃つゲーム | 返り咲き層・ギーク層に一瞬で伝わる | 非ギークには古い/狭い | 候補の一つとして見せる |
| スイカゲーム風 | スイカゲームみたいに、同じ果物をくっつけるゲーム | ルールが直感的。見た目が柔らかい | 物理挙動で沼る | 完全再現ではなくミニ版 |
| キャンディクラッシュ/パズドラ風 | キャンディクラッシュやパズドラみたいに、3つそろえるパズル | Match 3文脈。女性比率が高い既存ジャンルに近い | 消去・補充・連鎖で実装が重い | 5x5、4種、連鎖なしに削る |
| 好きなキャラでSnaky | Snaky Catみたいに、好きなキャラがアイテムを集めて長くなるゲーム | キャラ置換で自分ごと化しやすい。実装が安定 | データ根拠はMatch 3/Farmより弱い | 実演候補として強い |
| 小さな畑/収穫 | FarmVilleや牧場ゲームみたいに、水をあげて収穫するミニアプリ | Family/Farm Sim文脈。やさしい。実装が軽い | 映像の引きはゲームより弱い | 非ギーク向けの有力候補 |
| 献立ルーレット | 今日の夕飯を決めるルーレット | ゲームに興味ない層にも届く | 地味 | 冒頭の「ゲーム以外も作れる」例 |

## Recommended Opening

001はゲーム一本槍にしない。

冒頭で、複数の具体名風候補を高速に見せる。

```text
AIに頼むと、最初の1時間で小さいものが動きます。

たとえば、
インベーダーみたいなゲーム。
スイカゲームみたいな落ちもの。
キャンディクラッシュみたいな3つそろえるパズル。
FarmVilleみたいな小さな畑。
献立を決めるルーレット。

今日は、この中から一番「動いた」が分かりやすいものを作ります。
```

## How To Guide The User

初心者に「仕様を書け」と言わない。

こう導く。

```text
まず、知っているゲームやアプリの名前を出していい。
そこから、AIと一緒に小さい版に削る。
```

### Step 1: Name the familiar thing

```text
スイカゲームみたいなものを作りたい
キャンディクラッシュみたいなものを作りたい
FarmVilleみたいな小さい畑を作りたい
```

### Step 2: Shrink it

AIにこう聞く。

```text
最初の1時間で作れる、一番小さい版にしてください。
```

### Step 3: Decide the minimum

人間が決めるのはこれだけ。

```text
何を動かす？
何をしたら点が入る？
何が起きたら終わり？
最初は何を作らない？
```

### Step 4: Ask AI to clarify

AIにこう言う。

```text
作る前に、足りない情報があれば3つだけ質問してください。
```

### Step 5: Build

```text
まず動くところまで作って。
見た目の細かい調整は後で。
```

## Example Prompts

### スイカゲーム風ミニ版

```text
スイカゲームみたいに、同じ果物をくっつけるミニゲームを作りたい。
最初の1時間で作れる小さい版にしてください。
果物は3種類だけでいいです。
同じ果物がぶつかったら一段大きい果物になります。
箱からあふれたらゲームオーバーです。
作る前に、足りない情報があれば3つだけ質問してください。
```

### キャンディクラッシュ/パズドラ風ミニ版

```text
キャンディクラッシュやパズドラみたいに、かわいいアイコンを3つそろえるパズルを作りたい。
最初は5x5の盤面、アイコン4種類だけでいいです。
隣同士を入れ替えて、縦か横に3つ並んだら消えてスコアが増えます。
連鎖、特殊アイテム、制限時間はまだ作りません。
作る前に、足りない情報があれば3つだけ質問してください。
```

### 好きなキャラでSnaky

```text
Snaky Catみたいに、好きなキャラがアイテムを集めて長くなるゲームを作りたい。
キャラは後で差し替えたいので、最初は丸いアイコンでいいです。
アイテムを取るとスコアが増えて体が長くなります。
壁か自分の体にぶつかったらゲームオーバーです。
作る前に、足りない情報があれば3つだけ質問してください。
```

### 小さな畑/収穫アプリ

```text
FarmVilleみたいな、小さな畑アプリを作りたい。
最初は3つの鉢だけでいいです。
鉢をクリックすると水をあげられます。
3回水をあげると花になり、収穫するとスコアが増えます。
保存機能、ショップ、時間経過はまだ作りません。
作る前に、足りない情報があれば3つだけ質問してください。
```

### 献立ルーレット

```text
今日の夕飯を決めるルーレットアプリを作りたい。
候補を追加できます。
ボタンを押すとランダムで1つ選ばれます。
肉、魚、野菜のタグで絞り込めます。
保存機能は後でいいです。
作る前に、足りない情報があれば3つだけ質問してください。
```

## What To Avoid In 001

001で避けるもの。

```text
仕様
PRD
受け入れ基準
権利リスクの長い説明
商用利用の話
公式誤認の話
完全再現の話
細かいデザイン調整
mm単位のレイアウト
```

これらは重要だが、001のツカミではない。

## SNS / Public Sharing Note

最後に短く言う。

権利や公開リスクを主題にしない。
ただし、初心者がそのままSNSに上げて変にバズる可能性はゼロではない。

推奨文:

```text
今日は有名なゲームの型を借りて練習しました。
自分で遊ぶ・学ぶためのものです。
そのままSNSや公開サイトに上げるのはやめておきましょう。
見た目や名前を変えて、自分の作品にしてから見せるのが安全です。
```

もっと短くするなら:

```text
これは練習用です。
そのままSNSに上げず、名前や見た目を自分のものに変えてから見せましょう。
```

## Current Recommendation

001の最適構成:

```text
1. 具体名風候補を5つ見せる
2. 「ゲームだけじゃない」と言う
3. 今日は一番動いた感がある題材を1つ作る
4. AIに「足りない情報を3つ聞いて」と頼む
5. 小さい版へ削る
6. 動かす
7. 最後にSNS公開の注意を一言だけ添える
```

候補の優先順:

```text
1. 好きなキャラでSnaky
2. 小さな畑/収穫アプリ
3. スイカゲーム風ミニ版
4. キャンディクラッシュ/パズドラ風ミニ版
5. 献立ルーレット
6. インベーダー風
```

理由:

- Snakyは実装が安定し、キャラ置換で広げやすい。
- 畑/収穫はデータ上のFamily/Farm Sim文脈に近く、非ギークにやさしい。
- スイカゲーム風はフックが強いが、物理で沼る可能性がある。
- 3つそろえるパズルはデータ根拠が強いが、実装が重い。
- 献立ルーレットは非ゲーム層に届くが、映像の引きが弱い。
- インベーダー風は返り咲き層には強いが、入口としては狭い。
